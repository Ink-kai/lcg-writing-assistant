# 三环分析结论

## 环1：新杠杆 / 元杠杆

| 杠杆类型 | 可用性 | 具体分析 |
|----------|--------|----------|
| 代码杠杆 | ✅ 可用 | GitHub 开源项目可参考，WeChat API 可接入，多平台发布逻辑可标准化为插件功能 |
| 媒体杠杆 | ✅ 可用 | 一次写作多平台分发，用户越多，边际成本越低 |
| 元杠杆（AI） | ✅ 核心优势 | AI 可做 SEO/GEO/文章润色，自动格式转换（MD → HTML/平台特定格式），大幅降低适配成本 |

**结论**：杠杆密度高，适合做产品型工具。

---

## 环2：边界变动

### 技术边界变动
- AI 让内容格式适配从手工变成自动（以前需要人做，现在 AI 可以承担）
- vibe coding 让更多人有了个性化工作流，连接工具需求增加
- 各平台 API 逐步开放（微信、GitHub 等），接入成本降低

### 用户行为变化
- 技术博主开始用 Obsidian 等笔记工具管理工作流
- 多平台发布成为刚需（微信/X/小红书/知乎）
- 格式差异（MD vs HTML vs 各平台格式）是真实痛点

### 为什么大公司不来
- Notion 等大厂有多平台功能，但针对"技术博主 + Obsidian + Hugo"这个细分的适配不够
- 多平台维护成本高，大公司不愿意持续投入这种细分场景
- AI 辅助写作功能大公司有，但往往在付费墙后面

## 核心判断

**为谁**：使用 Obsidian 写 Hugo 博客的技术博主

**痛什么**：一次写作需要手动发布到多个平台，每个平台格式不同，图片/SEO 都要重复处理

**为什么有切口**：AI 让格式适配成为可能，市场还没有完整解决方案，你作为深度用户有独特优势

**为什么现在**：AI 工具成熟，vibe coding 普及，更多人有了个性化工作流，连接需求增加
