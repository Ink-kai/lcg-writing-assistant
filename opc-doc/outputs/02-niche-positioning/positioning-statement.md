# 定位陈述

## 主定位

```
为谁：使用 Obsidian 写 Hugo 博客的技术博主

解决什么问题：一次写作，多平台自动适配发布
（Obsidian MD → Hugo + 微信 + X + 小红书 + 知乎，自动处理格式/图片/SEO）

为什么是你：
- 自己是深度用户，清楚每个平台的坑
- 全栈开发，能从插件到 AI 工作流全部覆盖
- Claude Code 全天使用，迭代速度比大多数竞争者快

为什么现在：
- AI 让格式适配从不可能变成可能
- 市场还没有完整解决方案
- vibe coding 让更多人有了个性化工作流，连接需求增加
```

---

## 补充说明

### 目标用户特征
- 使用 Obsidian 管理笔记
- 使用 Hugo 构建博客
- 需要发布到多个平台（微信/X/小红书/知乎等）
- 技术背景，能自己配置工具
- 愿意为提升效率付费

### 核心价值
- **自动化**：减少重复性操作
- **格式适配**：自动处理平台差异
- **AI 辅助**：SEO/GEO/润色

### 用户约束
- 不愿主动经营人际关系 → 产品型工具
- 信息焦虑 → 避免重度信息输入工作

---

## 待验证假设

1. 技术博主群体有足够的付费意愿
2. "多平台发布"是真实强需求，不只是痒点
3. 可以通过社区（Obsidian forum, GitHub）触达目标用户
4. AI 辅助功能（SEO/GEO）能形成差异化
