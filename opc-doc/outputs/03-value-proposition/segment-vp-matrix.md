# 目标客群 × 价值主张对应关系

## 目标细分

**使用 Obsidian 写 Hugo 博客的技术博主**

---

## 价值主张矩阵

| 维度 | 内容 |
|------|------|
| **核心价值主张** | 提效型：让技术博主从重复发布中解放出来 |
| **对应的 Jobs** | 减少重复劳动，一次发布多平台同步 |
| **对应的 Pains** | 各平台 frontmatter 属性不统一，手动维护麻烦；SEO/元数据配置繁琐 |
| **对应的 Gains** | 一次发布，平台自动同步，无差异问题；节省时间，专注内容 |
| **Pain Relievers** | 自动转换 frontmatter 格式；一键发布；减少手动操作 |
| **Gain Creators** | 时间节省；工作流简化；全平台一致性 |

---

## 用户画像

- **使用场景**：在 Obsidian 里写文章，通过插件同步到 Hugo 和其他平台
- **核心诉求**：省时间，不折腾
- **付费意愿**：愿意为效率工具付费，但需要看到明确价值
- **技术能力**：能自己配置工具，不需要太多指导
- **决策因素**：操作简单、稳定可靠、覆盖主流平台

---

## 沟通要点

### 说什么（Do）
- "写一次，全平台同步"
- "不用手动调整格式和元数据"
- "从重复发布中解放出来"

### 不说什么（Don't）
- 不强调"AI SEO"等复杂功能（对技术博主来说不是核心诉求）
- 不强调"功能多"（他们要的是简单有效）
- 不使用"元数据转换"等技术术语（用"同步"更直观）
