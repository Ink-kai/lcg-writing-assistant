# 收费阶段与产品化路线

本文档定义 LCG Writing Assistant 从免费 Obsidian 插件走向付费产品的下一步路线。

## 产品判断

LCG Writing Assistant 不应该只做成“front matter 小工具”。更有价值的方向是：

**从 Obsidian 写作到 Hugo/FixIt 发布的工作流产品。**

免费版负责让用户在单篇文章里写得更快、更稳。付费版负责把批量处理、图片资产、发布检查、多站点发布这些高耗时流程自动化。

长期方向可以扩展为：

**Obsidian 内容发布中枢。**

先服务 Hugo/FixIt 用户，再扩展到多平台发布和 LLM wiki/知识库发布。

## 免费版边界

免费版目标：进入 Obsidian 市场，获得真实用户和反馈。

免费版应该继续包含：

- `/lcg` 写作入口
- front matter 模板插入
- front matter 可视化编辑
- 字段说明和模板字段配置
- 当前文章校验
- 单篇文章图片粘贴和手动上传
- 用户自己配置的 Cloudflare R2/WebDAV
- 上传测试
- 无遥测、无账号、默认本地优先

免费版不应该被削弱。它要能独立解决单篇文章写作问题。

## Pro 边界

Pro 版目标：出售“发布前后的省时能力”，不是出售基础字段编辑。

Pro 适合包含：

- 自动扫描 Hugo/FixIt 项目配置
- 从 archetypes、taxonomies、theme params 生成真实字段 schema
- 批量转换 Obsidian properties 到 Hugo front matter
- 批量图片上传、重命名、压缩、路径替换
- 孤儿图片和孤儿 CDN 对象检测
- 多站点配置
- 发布前检查：slug、draft、分类、标签、摘要、头图、图片可访问性、死链
- 一键发布准备：复制到 Hugo 项目、构建预览、Git commit
- 自定义校验规则
- 模板包和站点配置包

一句话边界：

**免费版是单篇文章编辑增强；Pro 版是 Hugo/FixIt 发布流水线。**

## 阶段 0：免费版收口

目标：让当前免费版适合长期留在 Obsidian 市场。

任务：

- 同步 README 和 README-zh 中的命令名、菜单名、设置说明。
- 保持隐私、网络请求、凭据存储说明清楚。
- 等真实赞助页准备好后再考虑 `fundingUrl`，不要提交占位链接。
- 修复 ObsidianReviewBot 的 required 问题。
- 不在免费版里加入强打扰的 Pro 弹窗。

验收：

- Obsidian 社区 PR 通过机器人检查。
- 免费版功能不需要账号、不需要联网授权。
- 用户能只靠 README 完成安装、配置、使用和隐私判断。

## 阶段 1：支持入口与路线图

目标：不急着卖 Pro，先建立“这个项目会长期维护”的信号。

任务：

- 在设置页底部加克制的 `支持项目` 区域。
- 准备真实赞助入口，例如 GitHub Sponsors、爱发电、Ko-fi。
- 写一页公开 Pro 路线图，说明未来付费功能聚焦批量和发布自动化。
- README 中增加“免费版和未来 Pro 边界”小节。

设置页入口建议：

```text
LCG Writing Assistant 是免费开源插件。
如果它节省了你的写作和发布时间，可以赞助项目或了解 Pro 路线图。
```

按钮：

- `赞助项目`
- `了解 Pro`

验收：

- 设置页没有弹窗、倒计时、灰色锁定广告。
- 用户能理解免费版不会被拆残。
- 赞助链接是真实可用链接。

## 阶段 2：Pro 早鸟验证

目标：验证是否有人愿意为发布工作流付费。

不要先做完整授权系统。先卖早鸟，手动交付。

建议价格：

- 国内早鸟：`¥49`
- 海外早鸟：`$9`

早鸟承诺只覆盖 2-3 个强痛点：

1. 扫描 Hugo/FixIt 项目并生成真实 schema。
2. 批量图片上传和 Markdown 路径替换。
3. 发布前检查和发布准备。

任务：

- 做一个早鸟说明页。
- 说明 Pro 早期功能、交付节奏、退款方式、隐私边界。
- 通过爱发电、Gumroad、Lemon Squeezy 或手动方式收款。
- 用手动 license key 或私有 release 交付。
- 收集 10-20 个早鸟用户的具体工作流。

验收：

- 至少 5 个真实用户表达明确付费意愿。
- 至少 3 个用户能说清楚“现在最浪费时间的发布步骤”。
- 早鸟用户接受手动交付，不强依赖自动授权。

## 阶段 3：Pro 最小可售版本

目标：交付第一个能独立收费的 Pro。

第一个 Pro 不要做大而全，聚焦 Hugo/FixIt 发布链路。

### 功能 1：Hugo/FixIt 项目扫描

能力：

- 读取 Hugo 项目目录。
- 识别 `config`、`hugo.toml/yaml/json`、theme params。
- 识别 archetypes。
- 识别 taxonomies。
- 生成字段 schema 和默认模板。

用户价值：

- 不再手动维护字段列表。
- 换主题或多站点时能快速适配。

### 功能 2：批量图片处理

能力：

- 扫描文章中的本地图片和 Obsidian 附件。
- 批量上传到用户配置的 R2/WebDAV。
- 生成稳定路径。
- 替换 Markdown 图片链接。
- 支持 dry run 预览。

用户价值：

- 从“手动搬图片、改路径”变成一次确认。
- 降低发布后图片失效风险。

### 功能 3：发布前检查

能力：

- 检查必填字段。
- 检查 draft、slug、date、lastmod。
- 检查分类、标签、摘要、头图。
- 检查图片 URL 可访问性。
- 检查本地链接和死链。
- 输出可修复问题列表。

用户价值：

- 发布前知道哪里会坏。
- 避免部署后再回头修文章。

验收：

- 一个真实 Hugo/FixIt 站点可以完成扫描、批量图片处理、发布前检查。
- 所有批量操作都有预览和撤销/备份策略。
- Pro 功能失败时不影响免费版基础编辑功能。

## 阶段 4：正式收费与授权

目标：减少人工发货成本，建立可维护收入流。

建议价格：

- 国内正式价：`¥99`
- 海外正式价：`$19`
- 团队/商业授权：`¥299-699`

授权策略：

- 免费功能永远不需要 license。
- Pro 功能需要 license key。
- license 本地保存。
- 校验成功后允许离线使用 30 天。
- 校验失败只锁 Pro，不影响免费功能。

授权请求只允许发送：

- license key
- 插件版本
- 随机安装 ID
- 可选邮箱

禁止发送：

- vault 内容
- 文件名
- front matter 内容
- 图片 URL 列表
- Cloudflare/WebDAV 凭据

验收：

- 用户能购买、输入 license、解锁 Pro。
- 离线宽限期可用。
- README 和设置页披露授权请求和隐私边界。

## 阶段 5：从插件到内容发布产品

目标：从 Hugo/FixIt 单点工具扩展为 Obsidian 内容发布中枢。

候选方向：

- 多平台发布版本生成：
  - Hugo
  - 微信公众号
  - 知乎
  - 掘金
  - Medium
  - Docusaurus
  - MkDocs
- 平台差异检查：
  - Markdown 方言
  - 图片托管
  - 摘要和标题长度
  - 外链规则
- LLM wiki/知识库发布：
  - 自动生成导航
  - 自动生成 topic map
  - 自动生成 glossary
  - 生成 `llms.txt`
  - 导出 RAG 友好的 Markdown/JSON bundle

判断标准：

- Hugo/FixIt 用户已经证明愿意为发布自动化付费。
- 有用户明确要求多平台分发或知识库发布。
- 插件架构已经能承载“扫描 -> 预览 -> 批量修改 -> 发布检查”的工作流。

## 近期执行顺序

1. 同步 README/README-zh 的当前 UI 文案。
2. 免费版通过 Obsidian 社区插件申请。
3. 准备真实赞助链接，决定是否加入 `fundingUrl`。
4. 给设置页加克制的 `支持项目` 区域。
5. 写公开 Pro 路线图页面。
6. 访谈或收集 5-10 个 Hugo/FixIt 写作者发布痛点。
7. 设计 Pro 早鸟页。
8. 开始 Pro MVP：项目扫描、批量图片处理、发布前检查。

## 不做清单

短期不做：

- 订阅制。
- 复杂账号系统。
- 强 DRM。
- 启动弹窗推广。
- 把基础编辑功能移到 Pro。
- 上传用户 vault 内容到作者服务器。
- 为收费牺牲免费版的本地优先和隐私透明。
